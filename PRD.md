# PRD: 직장 헬게이트 탈출 v3.0

**Product Requirements Document**
버전: 3.0 | 작성일: 2026-03-11 | 이전 버전: v2.0

---

## 1. 제품 개요

### 1.1 제품명
직장 헬게이트 탈출 (Workplace Hellgate Escape)

### 1.2 제품 비전
직장인이 일상에서 겪는 스트레스를 게임을 통해 재미있고 안전하게 해소할 수 있는 모바일 웹 탭 액션 게임

### 1.3 핵심 가치 제안
- 실제 직장 내 스트레스 유발자(빌런)를 직접 등록하고 두들겨 패며 카타르시스 제공
- 출퇴근, 점심시간 등 짧은 시간(5~10분)에 부담 없이 즐길 수 있는 캐주얼 경험
- 별도 설치 없이 모바일 웹 브라우저에서 즉시 플레이 가능

### 1.4 타겟 사용자
- **주요 타겟:** 직장인 (20~40대)
- **사용 맥락:** 출퇴근 대중교통, 점심시간 휴식, 야근 중 잠깐의 스트레스 해소

### 1.5 플랫폼 및 배포
- **플랫폼:** 모바일 웹 (PWA 형태)
- **배포:** Vercel (https://hell-escape.vercel.app)
- **기기 우선순위:** 모바일 스마트폰 (태블릿, 데스크톱 호환)

## 3. 사용자 흐름 (User Flow)

```
[설정 화면 (Setup)]
       ↓
  빌런 등록 & 닉네임 & 무기 선택
       ↓
[게임 화면 (Battle)]
       ↓
  탭 전투 → 빌런 피해 누적 시각화 → 빌런 격파
       ↓
[선택 이벤트 (Choice Modal)]  ← 마지막 빌런이 아닐 경우
       ↓
  다음 빌런 등장 → 전체 격파
       ↓
[결과 화면 (Result)]
       ↓
  점수 확인 → 리더보드 등록 → 다시 도전
```

## 4. 기능 요구사항

### 4.1 설정 화면 (Setup Screen)

#### 4.1.1 플레이어 닉네임
- 리더보드 등록에 사용되는 닉네임 입력 필드
- 필수 입력값 (비어있을 경우 게임 시작 불가)

#### 4.1.2 빌런 등록
- **빌런 유형 선택 (5종 프리셋):** 꼰대 상사 / 뒤통수 동료 / 무한 잔소리 / 야근 강요러 / 책임 전가러
- **빌런 이름 입력:** 직접 커스텀 (예: 김부장, 박과장)
- **빌런 특징 입력:** 메모 형식의 특징 입력
- **다중 빌런 등록 / 개별 삭제** 지원
- 최소 1명 이상의 빌런이 등록되어야 게임 시작 가능

#### 4.1.3 무기 선택 (신규 — FB-003)
게임 시작 전 무기 1종을 선택. 선택한 무기는 게임 전체에 적용.

| 무기 | 아이콘 | 공격 특성 | 타격 이펙트 |
|------|--------|-----------|------------|
| 망치 | 🔨 | 중타격, 느린 콤보 | 멍·골절 시각화 |
| 칼 | 🔪 | 빠른 다단 타격 | 출혈·살점 시각화 |
| 총 | 🔫 | 원거리, 구멍 누적 | 탄흔 누적 시각화 |
| 불 | 🔥 | 지속 피해, 광역 | 불꽃 번짐 시각화 |

- 무기 선택 UI: 설정 화면 하단 4-grid 버튼
- 선택하지 않으면 기본 무기(주먹 👊) 적용

#### 4.1.4 리더보드 접근
- 설정 화면에서 현재 상위 10개 기록(명예의 전당) 확인 가능

### 4.2 게임 화면 (Battle Screen)

#### 4.2.1 상단 HUD

| 게이지 | 설명 | 동작 |
|--------|------|------|
| ❤️ HP 게이지 | 현재 빌런의 체력 | 공격 시 감소, 0이 되면 격파 |
| 😤 스트레스 게이지 | 플레이어의 스트레스 수치 | 공격할수록 감소 |
| ⚡ 필살기 게이지 | 필살기 충전량 | 공격/콤보 시 충전, 100%에서 사용 가능 |

#### 4.2.2 전투 시스템

| 기능 | 상세 사양 |
|------|----------|
| **탭 공격** | 빌런 탭/클릭 시 무기별 데미지 범위 적용 |
| **콤보 시스템** | 연속 탭 시 콤보 카운트 증가 → 데미지 배율 상승 |
| **콤보 초기화** | 마지막 탭 후 1.6초 경과 시 콤보 카운트 리셋 |
| **필살기** | 게이지 100% 충전 후 필살기 버튼 탭 → 대형 피해 + 화려한 이펙트 |
| **빌런 감정 변화** | HP 잔량에 따라 이모지 변화: 😤 → 😰 → 😱 → 🤮 → 💀 |
| **다중 빌런 순환** | 현재 빌런 격파 시 선택 이벤트 → 다음 빌런 등장 |

**무기별 데미지 범위:**

| 무기 | 기본 데미지 | 콤보 보너스 | 특수 효과 |
|------|------------|------------|-----------|
| 주먹 👊 | 4~11 | +1.5×콤보 | 없음 (기존 동작 유지) |
| 망치 🔨 | 8~18 | +2×콤보 | 콤보 5↑시 기절(0.5초 추가 데미지) |
| 칼 🔪 | 3~7 (×3회) | +1×콤보 | 출혈: 1초마다 2 추가 데미지 (3초) |
| 총 🔫 | 10~20 | +0.5×콤보 | 탄흔 누적 (시각 효과) |
| 불 🔥 | 5~10 | +3×콤보 | 점화: 2초마다 5 지속 피해 (5초) |

#### 4.2.3 피해 누적 시각화 (신규 — FB-001, FB-002)

빌런 이모지 주변에 피해 레이어를 CSS 오버레이로 표현. HP 감소량에 따라 단계적으로 추가됨.

**무기별 피해 오버레이:**

| 무기 | 25% HP 손실 | 50% HP 손실 | 75% HP 손실 | 90%+ HP 손실 |
|------|------------|------------|------------|--------------|
| 망치 🔨 | 멍 이모지 💜×1 | 💜×3 + 🩹 | 골절 표시 💥 | 전신 멍 + 흔들림 강화 |
| 칼 🔪 | 상처선 (CSS border) | 빨간 점 🔴×2 | 붉은 오버레이 | 피투성이 플래시 |
| 총 🔫 | 탄흔 ⚫×1 | ⚫×3 | ⚫×6 (너덜너덜) | ⚫×10 + 생선가시 CSS |
| 불 🔥 | 불꽃 테두리 | 주황 글로우 강화 | 검은 그을림 | 재(ash) CSS 필터 |

- 오버레이는 빌런 이모지를 감싸는 `position: relative` 컨테이너에 `::after` 또는 절대 위치 자식 요소로 구현
- HP 단계가 바뀔 때마다 `damageStage` state(0~4) 업데이트 → 해당 단계의 클래스 추가

#### 4.2.4 선택 이벤트 (신규 — FB-005)

빌런 격파 후 다음 빌런 등장 직전(마지막 빌런 제외) 도덕적 딜레마 모달 표시.

**모달 구성:**
- 격파한 빌런 정보 표시
- 행동 선택 3종:

| 선택지 | 아이콘 | 점수 효과 | 다음 빌런 효과 |
|--------|--------|-----------|--------------|
| 마무리 | ⚡ "이대로 끝낸다" | +100점 | 없음 |
| 구호품 전달 | 📦 "구호품을 보낸다" | -50점 | 다음 빌런 HP -20 (약해짐) |
| 치료 지원 | 💊 "치료를 지원한다" | +200점 | 다음 빌런 HP +30 (강해짐) |

- 선택 후 3초 내 자동 진행 (타이머 표시)
- 선택하지 않으면 "마무리" 자동 선택
- 결과 화면에 선택 이력 요약 표시

#### 4.2.5 시각 이펙트 (기존 + 확장)

| 이펙트 | 사양 |
|--------|------|
| 플로팅 텍스트 | 무기별 타격 문구 + 콤보 수 표시 |
| 충격파 애니메이션 | 무기별 색상·크기 변형 (망치=노랑, 칼=빨강, 총=흰색, 불=주황) |
| 화면 흔들림 | 무기별 진폭 차등 (망치 > 주먹 > 칼 > 불 > 총) |
| 배경 플래시 | 무기별 플래시 색상 변경 |
| 피해 누적 오버레이 | HP 단계별 시각 피해 표현 (신규) |

**무기별 타격 문구 확장:**

## 5. 비기능 요구사항

### 5.1 성능
- 첫 로드 후 게임 시작까지 2초 이내
- 탭 입력 반응 지연 100ms 이하
- 피해 오버레이 추가로 인한 렌더링 지연 없음 (CSS transition 활용)

### 5.2 접근성 및 UX
- 한 손 세로 모드 조작 최적화
- 음소거 기능 기본 제공
- 진동 미지원 기기에서도 정상 동작
- 선택 이벤트 모달: 타이머 3초 + 자동 진행으로 답답함 방지

### 5.3 브라우저 호환성
- iOS Safari, Android Chrome 최신 2개 메이저 버전 지원
- Web Audio API, Vibration API, localStorage 사용 (미지원 시 graceful degradation)
