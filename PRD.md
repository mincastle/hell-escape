# PRD: 직장 헬게이트 탈출 v2.0

**Product Requirements Document**
버전: 2.0 | 작성일: 2026-03-11

---

## 1. 제품 개요

### 1.1 제품명
직장 헬게이트 탈출 (Workplace Hellgate Escape)

### 1.2 제품 비전
직장인이 일상에서 겪는 스트레스를 게임을 통해 재미있고 안전하게 해소할 수 있는 모바일 웹 탭 액션 게임

### 1.3 핵심 가치 제안
- 실제 직장 내 스트레스 유발자(빌런)를 직접 등록하고 두들겨 패며 카타르시스 제공
- 출퇴근, 점심시간 등 짧은 시간(5~10분)에 부담 없이 즐길 수 있는 캐주얼 경험
- 별도 설치 없이 모바일 웹 브라우저에서 즉시 플레이 가능

### 1.4 타겟 사용자
- **주요 타겟:** 직장인 (20~40대)
- **사용 맥락:** 출퇴근 대중교통, 점심시간 휴식, 야근 중 잠깐의 스트레스 해소

### 1.5 플랫폼 및 배포
- **플랫폼:** 모바일 웹 (PWA 형태)
- **최종 목표:** 앱인토스(Appintos) 출시
- **기기 우선순위:** 모바일 스마트폰 (태블릿, 데스크톱 호환)

## 2. 성공 지표 (Success Metrics)

| 지표 | 목표 |
|------|------|
| 세션 평균 플레이 시간 | 5~10분 |
| 재방문율 (7일 리텐션) | 30% 이상 |
| 리더보드 등록율 | 전체 플레이어의 50% 이상 |
| 음소거 사용율 | 측정 후 UX 개선 반영 |

---

## 3. 사용자 흐름 (User Flow)

```
[설정 화면 (Setup)]
       ↓
  빌런 등록 & 닉네임 입력
       ↓
[게임 화면 (Battle)]
       ↓
  탭 전투 → 빌런 격파 → 다음 빌런 등장 → 전체 격파
       ↓
[결과 화면 (Result)]
       ↓
  점수 확인 → 리더보드 등록 → 다시 도전
```

## 4. 기능 요구사항

### 4.1 설정 화면 (Setup Screen)

#### 4.1.1 플레이어 닉네임
- 리더보드 등록에 사용되는 닉네임 입력 필드
- 필수 입력값 (비어있을 경우 게임 시작 불가)

#### 4.1.2 빌런 등록
- **빌런 유형 선택 (5종 프리셋)**
  - 꼰대 상사
  - 뒤통수 동료
  - 무한 잔소리
  - 야근 강요러
  - 책임 전가러
- **빌런 이름 입력:** 직접 커스텀 (예: 김부장, 박과장)
- **빌런 특징 입력:** 메모 형식의 특징 입력 (예: 항상 야근 강요, 뒤통수의 달인)
- **다중 빌런 등록:** 여러 빌런 추가 가능
- **개별 삭제:** 등록된 빌런 각각 삭제 가능
- 최소 1명 이상의 빌런이 등록되어야 게임 시작 가능

#### 4.1.3 리더보드 접근
- 설정 화면에서 현재 상위 10개 기록(명예의 전당) 확인 가능

---

### 4.2 게임 화면 (Battle Screen)

#### 4.2.1 상단 HUD

| 게이지 | 설명 | 동작 |
|--------|------|------|
| ❤️ HP 게이지 | 현재 빌런의 체력 | 공격 시 감소, 0이 되면 격파 |
| 😤 스트레스 게이지 | 플레이어의 스트레스 수치 | 공격할수록 감소 |
| ⚡ 필살기 게이지 | 필살기 충전량 | 공격/콤보 시 충전, 100%에서 사용 가능 |

#### 4.2.2 전투 시스템

| 기능 | 상세 사양 |
|------|----------|
| **탭 공격** | 빌런 이모지 탭/클릭 시 랜덤 5~12 데미지 |
| **콤보 시스템** | 연속 탭 시 콤보 카운트 증가 → 데미지 배율 상승 |
| **콤보 초기화** | 마지막 탭 후 1.6초 경과 시 콤보 카운트 리셋 |
| **필살기** | 게이지 100% 충전 후 필살기 버튼 탭 → 40~60 데미지 + 화려한 이펙트 |
| **빌런 감정 변화** | HP 잔량에 따라 이모지 변화: 😤 → 😰 → 😱 → 🤮 → 💀 |
| **다중 빌런 순환** | 현재 빌런 격파 시 다음 등록 빌런 자동 등장 |
| **음소거 버튼** | 🔊/🔇 토글 (지하철 등 공공장소 배려) |

#### 4.2.3 시각 이펙트

| 이펙트 | 사양 |
|--------|------|
| 플로팅 텍스트 | 타격 위치에 랜덤 타격 문구 + 현재 콤보 수 표시 |
| 충격파 애니메이션 | 탭 위치에서 원형 충격파 확산 |
| 화면 흔들림 | 타격마다 shake 효과 적용 |
| 배경 플래시 | 타격 시 배경색 순간 변화 |

---

### 4.3 결과 화면 (Result Screen)

#### 4.3.1 표시 통계

| 항목 | 계산 방식 |
|------|----------|
| 총 점수 | 공격 횟수×10 + 최대 콤보×50 + 격파 빌런 수×200 + 스트레스 해소율×5 |
| 총 공격 횟수 | 플레이 중 전체 탭 횟수 합산 |
| 최대 콤보 | 게임 중 기록한 최고 콤보 수 |
| 격파 빌런 수 | 처치한 빌런 총 인원 |
| 스트레스 해소율 | 게임 중 감소한 스트레스 수치의 % |

#### 4.3.2 리더보드 등록
- 게임 종료 후 닉네임과 점수를 로컬 리더보드에 자동 저장
- 상위 10개 기록만 유지

#### 4.3.3 액션 버튼
- **다시 도전:** 설정 화면으로 돌아가기

---

### 4.4 사운드 시스템

| 사운드 | 구현 방식 | 상세 |
|--------|----------|------|
| 배경음악 | Web Audio API 비트 루프 | 외부 파일 불필요, 자체 생성 |
| 타격음 | Web Audio API | 콤보 수에 따라 음높이·강도 상승 |
| 필살기 사운드 | Web Audio API | 6단계 하강 폭발음 |
| 격파 사운드 | Web Audio API | 4음 상승 팡파레 |

- 음소거 상태는 세션 내 유지
- 모든 사운드는 외부 파일 의존 없이 Web Audio API로 프로그래매틱 생성

---

### 4.5 진동 피드백 (모바일 전용)

| 상황 | 진동 패턴 |
|------|----------|
| 일반 타격 | 짧게 1회 (20ms) |
| 콤보 5↑ 타격 | 더블 진동 (30ms - 10ms 간격 - 30ms) |
| 필살기 | 강한 패턴 (100-50-100-50-200ms) |
| 빌런 격파 | 5연속 패턴 (50-30-50-30-100ms) |

- Vibration API 미지원 기기에서는 진동 없이 정상 동작

---

### 4.6 리더보드

| 항목 | 사양 |
|------|------|
| 저장 방식 | `localStorage` (기기 내 로컬 저장) |
| 저장 용량 | 상위 10개 기록만 유지 (초과 시 최하위 기록 제거) |
| 표시 항목 | 순위 / 닉네임 / 점수 / 날짜 |
| 접근 시점 | 설정 화면 및 결과 화면 양쪽에서 확인 가능 |

## 5. 비기능 요구사항

### 5.1 성능
- 첫 로드 후 게임 시작까지 2초 이내
- 탭 입력 반응 지연 100ms 이하
- 외부 리소스(이미지, 폰트, 사운드 파일) 최소화 → 오프라인·저속 환경 대응

### 5.2 접근성 및 UX
- 한 손 세로 모드 조작 최적화
- 음소거 기능 기본 제공 (지하철 등 공공장소 배려)
- 진동 미지원 기기에서도 정상 동작

### 5.3 브라우저 호환성
- iOS Safari, Android Chrome 최신 2개 메이저 버전 지원
- Web Audio API, Vibration API, localStorage 사용 (미지원 시 graceful degradation)
